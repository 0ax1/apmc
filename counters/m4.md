# Apple M4 PMC Events

Source: `/usr/share/kpep/cpu_100000c_2_17d5b93a.plist`

## Fixed Counters

Always active, not programmable.

| Slot | Name | Alias |
|------|------|-------|
| 0 | FIXED_CYCLES | Cycles |
| 1 | FIXED_INSTRUCTIONS | Instructions |

## Configurable Events (101)

8 counter slots available (kernel reserves 3 until `force_all_ctrs_set(1)`). Events with a `counters_mask` can only be assigned to specific slots.

### Slot Constraints

- **any slot** -- can be assigned to slots 0-7
- **mask=0xfc** -- restricted to slots 2-7
- **mask=0x80** -- restricted to slot 7 only

## Notes

- **Speculative vs Non-speculative**: Events ending in `_NONSPEC` count only architecturally retired operations. Events without that suffix may include speculative operations that were later squashed.
- **counters_mask**: The kernel rejects events programmed into invalid slots. `apmc` handles slot assignment automatically.
- **Slot reservation**: The kernel reserves 3 of 8 configurable counter slots by default. `apmc` reclaims them via `kpc_force_all_ctrs_set(1)`.
- **SME**: The M4 adds Scalable Matrix Extension events not present on M2. These cover the AMX/SME engine used for matrix and ML workloads.
- **Arm TopDown**: The M4 exposes `ARM_STALL_SLOT*` events for slot-level frontend/backend stall analysis, matching the Arm TopDown methodology.
- **Per-process**: `apmc stat` measures counters for the target process and its descendants, not system-wide. Background system activity does not contribute noise.
- **Root required**: Programming configurable counters requires `sudo`. The `apmc list` command works without root.
